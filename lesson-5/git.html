Git and Github

Using a Version Control System (VCS) to manage your code is an important thing to learn how to do. Git is software that runs on your computer to manage code locally, while Github is a Web application that enables users to host their code in the cloud. Managed code bases are called repositories. Today, we will set up a local repository with Git, and then connect it with a remote repository on Github.

Create a Local Repository

In the Terminal, execute the following commands to create a local code repository:

  1. git init (Initializes the current folder as a local repository)
  2. git add . (Adds or stages all the files and folders in the current directory in preparation to commit)
  3. git commit -m "[short description here]" (Commits the new files to the repository)

Create a Remote Repository

Next, we will need to create a remote repository to push the local repository to:

  1. Proceed to Github and log in
  2. Create a new public or private repository called comp1054-lesson-4 or something like that - don't add a README.md file - we will do this later

Push the Local Repository to the Remote

Finally, we need to add the remote repository as the origin for your local repository:

  1. In the VSCode terminal, type git remote add origin https://github.com/[your_username]/[your_repository_name] (the URL ends with .git)
  2. git branch -M main (sets the current active branch to 'main')
  3. Upload the files and folders in your local repository to the remote with git push -u origin main
  4. Go back to Github and refresh the remote repository page to verify the push worked.

Learn more about working with Git at https://git-scm.com/.
